# Configuration Files

This directory contains configuration files that control pipeline behavior, parameters, and execution settings.

## Configuration Files

- `config.yaml`  
  Primary configuration file used during pipeline execution.

- `config_param_example.yaml`  
  Example configuration illustrating expected parameter structure and formatting.

- `config_parameter.yaml`  
  Parameter definitions and defaults used to customize pipeline components.

## Notes

- Configuration files define thresholds, tool options, and workflow settings.
- Unless otherwise specified, parameters largely reflect default or recommended values.
- These files are included to support transparency and reproducibility.
